# Local Notice Radar

一个本地运行的 localhost 信息收集页面，用于抓取并归档多个政府网站通知公告。

## 运行方式

```bash
npm install
npm run dev
```

打开：

```text
http://localhost:3333
```

## 当前 MVP 功能

- 右上角点击“运行”后，抓取已配置网站的公告列表。
- 默认每天 08:00 和 14:00 自动抓取，时间可在页面顶部修改。
- 自动筛选当前选择日期的公告。
- 抓取结果会保存到本地 `data/notices.json`。
- 默认保留今日往前 30 天的公告；手动抓取 30 天以前的日期时，该日期公告保留 1 天后自动清理。
- 可以通过日期选择器查看某一天已存储的公告。
- 点击日期选择器会显示已归档日期和对应公告数量。
- 公告列表支持导出 Word 或 PDF，默认导出到下载目录，导出路径会记住上次设置。
- 如果网站抓取失败，或某个网站在所选日期没有解析到公告，会在页面用红色提示。
- 右侧“站点管理”支持添加和编辑网页配置。
- 编辑站点时可以删除站点配置；删除不会清理已归档公告。
- 新增或编辑的网站会保存到本地 `data/sites.json`。
- 最近一次运行中抓取失败的网站会显示为“已失效”。

## 站点配置说明

- 网页地址：浏览器打开时展示和跳转使用的地址。
- 抓取地址：可选。如果某些网站 HTTPS 在 Node.js 中不兼容，可以填 HTTP 地址用于爬虫抓取。
- 标题关键词：用于过滤公告标题，多个关键词可用中文逗号或英文逗号分隔。
- 抓取页数：用于自动抓取分页。常见政府网站会按 `index.html`、`index_2.html`、`index_3.html` 分页，填 2 或 3 后会自动合并多页结果。
